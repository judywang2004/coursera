                          I've used both Mongo and SQL databases such as MySQL and PostgreSQL. Both have their purposes, but I often find myself using Mongo despite my data being highly relational. The reason is this: if I want to add a new feature to an application I'm developing, I have to redesign the schema, models, and controllers to account for that feature. Mongo, I just have to create a new route and a new function in the controller to handle that new functionality, so I and my team can iterate new features faster. That said, it's important to know both. Mongo and SQL

                              Worth noting that it's not always as simple as running the installer to update. Sometimes you'll have package problems, etc. If you do run "npm update -g" in cmd.

                              And if you are having serious problems run "npm cache clean" in cmd. Once the cache is empty, just run npm update -g again.
